# Perguntas 

**1. O que são operadores em programação?**
Operadores são símbolos usados para realizar operações com valores e variáveis.

**2. Para que servem os operadores aritméticos?**
Os operadores aritméticos servem para realizar cálculos matemáticos.

**3. Qual a diferença entre / e % em JavaScript?**
O operador / realiza a divisão, enquanto % retorna o resto da divisão.

**4. Como descobrir se um número é par usando o operador %?**
Um número é par quando o resto da divisão dele por 2 é igual a 0.

**5. O que é uma expressão em programação?**
Uma expressão é uma combinação de valores, variáveis e operadores que produz um resultado.

**6. O que significa precedência de operadores?**
Precedência de operadores é a ordem em que as operações são executadas em uma expressão.

**7. Por que os parênteses podem mudar o resultado de uma expressão?**
Porque as operações dentro deles são executadas primeiro.

**8. Qual a diferença entre =, == e ===?**
= atribui um valor; == compara valores; === compara valores e tipos.

**9. Por que é recomendado usar === em JavaScript?**
Porque ele faz comparações mais seguras e precisas.

**10. O que os operadores relacionais retornam: texto, número ou boolean?**
Retornam valores booleanos (true ou false).

**11. Quando usamos o operador >? E o operador >=?** >é usado para verificar se um valor é maior que outro; >= verifica se é maior ou igual.

**12. Quando usamos o operador <? E o operador <=?**
< é usado para verificar se um valor é menor que outro; <= verifica se é menor ou igual.

**13. Para que serve o operador lógico &&?**
Serve para verificar se duas ou mais condições são verdadeiras ao mesmo tempo.

**14. Para que serve o operador lógico ||?**
Serve para verificar se pelo menos uma das condições é verdadeira.

**15. Para que serve o operador lógico !?**
Serve para inverter o valor lógico de uma condição.

**16. Qual a diferença entre uma condição simples e uma condição composta?**
Uma condição simples possui apenas uma condição; uma condição composta possui duas ou mais condições.

**17. Explique uma situação real em que duas condições precisam ser verdadeiras ao mesmo tempo.**
Quando uma ação depende do cumprimento simultâneo de dois requisitos.

**18. Explique uma situação real em que apenas uma entre duas condições precisa ser verdadeira.**
Quando qualquer uma das condições já é suficiente para permitir uma ação.

**19. Como o console.log() pode ajudar a encontrar erros no código?**
Ele ajuda a visualizar informações do programa e identificar possíveis erros.

**20. Por que devemos testar o mesmo programa com valores diferentes?** Para verificar se ele funciona corretamente em diversas situações.
